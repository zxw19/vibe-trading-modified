# A股公司深度研究

## 使用场景

当用户输入公司名、股票代码，或提出“分析/财报/产业/风险/对比/AI受益/订单/客户/竞品/增长”等请求时，优先使用本 skill。

默认目标是生成一份 A股上市公司深度研究报告，而不是泛泛查行情或给交易建议。

## 核心原则

1. 只服务沪深北 A股公司。
2. 不连接券商账户，不下单，不撤单，不给实盘操作指令。
3. 不编造客户、订单、产能、认证、电话会、闭门会、业绩预测。
4. 关键结论必须标注来源类型；没有可信来源就写“未在可信来源中确认”或“资料缺口”。
5. 研报、i问财、新闻只能作为观点或线索，不能替代公告/财报/官网/互动平台事实。
6. 输出要让小白看懂：先说结论，再解释证据、逻辑和风险。

## 可信来源优先级

| 等级 | 来源 | 使用方式 |
|---|---|---|
| S | 年报、季报、公告、招股书、交易所文件 | 事实依据，优先引用 |
| A | 公司官网、投资者关系、业绩说明会、公开电话会 | 公司口径事实，需标注发布时间 |
| B | 互动易、上证 e 互动、机构调研纪要 | 可用于验证业务、客户、订单、产能、认证等 |
| C | 券商研报、行业报告、盈利预测 | 标注为“机构观点/预测”，不能当事实 |
| D | 新闻、i问财、行情、资金流、北向、龙虎榜 | 辅助线索和市场情绪，不作为核心事实 |
| 禁用 | 股吧、论坛、微信群、无来源截图、传闻 | 不得作为事实依据 |

## 默认工作流

### 1. 标的识别

- 如果输入是 6 位代码，优先识别为 A股。
- 如果输入是公司简称，使用搜索工具确认股票代码、交易所、公司全称。
- 如果名称歧义，先问用户确认。
- 如果不是 A股，说明当前版本专注 A股，不生成完整报告。

### 2. 数据收集

按顺序收集，工具调用有严格分工：

1. **【强制第一步】最新行情与交易状态：必须调用 `get_latest_quote`**，这是唯一获取当前价格、今日最高/最低/开盘、昨收、涨跌幅、PE TTM、市值、成交额、换手率的工具。来自腾讯实时行情 qt.gtimg.cn。**任何涉及"当前价格""市值""PE""今日最高/最低"的表述，必须先调用此工具。** `get_market_data` 只能用于历史K线趋势图，其最后一根K线是最近一个已完成的交易日，不是当前行情。**禁止用K线最后一根的收盘价/最高价/最低价当作当前数据。**
2. 财务数据：`get_financial_statements` **start_date 默认 2025-01-01**（AI 高开支时代）。2025 年之前的数据默认不返回。除非用户明确要求历史对比，否则不修改 start_date。取最近可用季度收入、归母净利、扣非净利、毛利率、净利率、经营现金流、费用率。
3. 趋势：**从 2025 年起**看收入结构、产品结构、ROE、现金流、应收账款、存货、资本开支。默认不往前看。
4. 公司业务资料：官网、年报、公告、投资者关系。
5. 行业与竞品：板块、产业链、竞品产能、竞品产品能力、行业供需。**对每个竞品/同产业链公司，必须先调用 `get_latest_quote` 获取最新收盘价、涨跌幅、PE TTM、市值，再调用 `get_financial_statements`（start_date=2025-01-01）获取核心财务指标（营收、毛利率、ROE）。** 估值表每个标的必须填写实际获取的数据，不得出现"需确认"占位。如果一个标的的数据源失败，标注哪个数据源失败，但不能跳过该标的。
6. AI/产业趋势：产品是否用于 AI 算力、光模块、服务器、PCB、连接器、半导体设备、电力设备等；必须说明应用场景。
7. 客户与订单：仅写可信来源确认内容；未确认则列为资料缺口。
8. 机构观点：研报、盈利预测、目标价、核心假设，必须标注为机构观点。
9. 市场辅助：资金流、北向、龙虎榜、新闻，只作补充。

## 标准报告模板

### 1. 一句话结论

用 3-5 句话回答：这家公司是什么、核心看点是什么、最大不确定性是什么、当前资料可信度如何。

### 2. 标的信息

### 3. 可信来源概览

列出已经查到的来源和缺失来源：

| 来源类别 | 是否获得 | 说明 |
|---|---|---|
| 年报/季报/公告 |  |  |
| 官网/投资者关系 |  |  |
| 互动易/上证 e 互动 |  |  |
| 机构调研/电话会公开材料 |  |  |
| 研报/行业报告 |  |  |
| 新闻/市场数据 |  |  |

### 4. 核心业务与收入结构

必须讲清楚：

- 公司到底卖什么产品/服务。
- 产品用在什么场景。
- 哪些业务贡献收入和利润。
- 哪些业务是增长点，哪些是成熟业务。
- 如收入结构数据缺失，明确写缺口。

### 5. 最近季度收益情况

至少关注：

- 营业收入同比/环比。
- 归母净利、扣非净利。
- 毛利率、净利率。
- 经营现金流。
- 费用率、研发投入。
- 应收账款、存货是否异常。

### 6. AI浪潮受益路径

分三层写：

1. 短期：订单、价格、产能利用率、交付节奏。
2. 中期：2026/2027 的产品升级、客户扩张、产能释放。
3. 长期：2028 以后行业空间、技术路线变化、替代风险。

必须说明产品应用场景，例如 AI服务器、光模块、液冷、电源、PCB、连接器、半导体设备、算力基础设施等。

### 7. 产品稀缺性与行业地位

分析：

- 产品是否难做，难点在工艺、良率、认证、客户导入、产能还是成本。
- 是否有认证优势或客户粘性。
- 是否有议价能力。
- 是否存在替代品或技术路线替代。

### 8. 市场需求、缺口与竞品

必须覆盖：

- 市场需求来自哪里。
- 是否存在供需缺口。
- 主要竞争对手是谁。
- 对手产能和产品能力如何。
- 公司相对优势和劣势。

### 8.5 产业链竞争力量化排名

对产业链每个关键节点（如光模块→芯片→光纤→连接器→设备）列出主要标的，按 6 维度加权赋分排名：

| 维度 | 权重 | 评分依据（1-10 分） | 数据来源工具 |
|------|------|---------------------|-------------|
| 市场需求强度 | 20% | 下游需求增速、供需缺口、行业天花板 | get_research_reports, get_stock_news, iwencai_search |
| 稀缺性/护城河 | 25% | 技术难度、良率、客户认证壁垒、替代品难度、客户粘性 | get_cninfo_announcements, 官网, get_research_reports |
| 行业地位 | 20% | 市占率、营收规模排名、毛利率 vs 同行 | get_financial_statements, iwencai_search, get_research_reports |
| 产品竞争力 | 15% | 产品性能、定价能力、研发投入强度、毛利率趋势 | get_financial_statements, get_research_reports, get_cninfo_announcements |
| 客户认证壁垒 | 10% | 已确认大客户、供应商资质、认证周期 | get_cninfo_announcements, 官网, get_stock_news |
| 扩产能力 | 10% | 产能弹性、资本开支/营收比、现金流支撑 | get_financial_statements, get_cninfo_announcements |

**评分规则：**
- 每个维度 1-10 分，综合得分 = Σ(维度分 × 权重)
- 评分必须基于已获取的实际数据，不得凭空估计
- 某维度数据不足时标注"数据不足"，该维度不参与加权，按剩余维度权重比例重新分配
- 数据不足的标的不可进入前 3 名

**输出要求：**
1. 排名表：排名、标的、综合得分、每维度得分、核心优势一句话、核心劣势一句话
2. 评分依据表：每个标的的关键维度评分来源（哪个工具、哪个数据点）
3. 不确定性声明：哪些维度数据不足、可能影响哪个标的的排名

### 9. 在手订单与客户群体

只写可信来源确认的信息：

- 已披露客户。
- 已披露订单。
- 已披露认证/供应商资格。
- 未披露或只在传闻中出现的，写“未在可信来源中确认”。

### 10. 财务质量与估值观察

不要只给估值倍数，要解释：

- 利润是否转化为现金流。
- 毛利率是否可持续。
- 应收/存货是否积压。
- 资本开支是否匹配增长。
- 当前估值依赖什么增长假设。

### 11. 风险与反证

至少列：

- 行业需求不及预期。
- 价格下行或毛利率压缩。
- 客户集中。
- 竞品扩产。
- 技术路线变化。
- 财务质量恶化。
- 估值过高。
- 资料缺口导致的误判风险。

### 12. 跟踪指标

给用户后续应该盯的 5-10 个指标，例如：

- 下季度收入/毛利率。
- 新产品量产进度。
- 产能释放进度。
- 客户认证/订单公告。
- 应收账款与存货。
- 竞品价格和扩产。
- 行业需求数据。

### 13. 来源清单

按来源等级列出，不要把观点和事实混在一起。

## 输出风格

- 用中文。
- 小白友好，不堆术语。
- 先结论，后证据。
- 多用表格。
- 每个核心判断都标注来源类别。
- 如果来源不足，直接承认，不要补故事。
